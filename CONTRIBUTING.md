# Contributing to NODEX

## Filosofia

Prima di scrivere codice, progettare.

Prima di progettare, comprendere il problema.

NODEX nasce come framework orientato alla conoscenza.

Ogni nuova classe deve avere una responsabilità precisa.

Il Core deve rimanere piccolo, stabile e indipendente.

Le funzionalità specifiche appartengono ai Modules.

La conoscenza appartiene ai Node e alle Relation.

## Regole di sviluppo

1. Una classe deve avere una sola responsabilità.
2. Ogni nuova classe deve superare il controllo `php -l`.
3. Ogni modifica significativa deve essere accompagnata da un commit Git descrittivo.
4. Prima di aggiungere una funzionalità verificare che non esista già una soluzione nel Core.
5. Il codice deve essere leggibile prima ancora che intelligente.
6. Le decisioni architetturali devono essere documentate in `DECISIONS.md`.
7. Ogni nuova versione aggiorna anche `CHANGELOG.md` e `ROADMAP.md`.

## Motto

La conoscenza non si archivia.
Si collega.
