# NODEX Architecture

## Motto

La conoscenza non si archivia. Si collega.

## Principio guida

NODEX non memorizza semplicemente informazioni.
NODEX costruisce relazioni tra conoscenze.

## Visione

NODEX è un Memory Operating System pensato per gestire entità della conoscenza chiamate Nodi.

La prima applicazione costruita su NODEX è:

Archivio Centrale dei Custodi del Doge.

## Separazione dei livelli

### NODEX Core

Il Core non conosce i Custodi del Doge, Venezia, la Riviera del Brenta o i Dossier.

Il Core conosce solo:

- Nodi
- Relazioni
- Configurazione
- Registro
- Storage
- Eventi
- Permessi

### Archivio Centrale

L’Archivio Centrale usa NODEX per gestire:

- Dossier
- Ville
- Luoghi
- Personaggi
- Documenti
- Reperti
- Bibliografia
- Galleria
- Timeline
- Mappa
- Confraternita

## Nodo

Un Nodo è una Entità della Conoscenza.

Ogni Nodo possiede sempre:

- UUID NODEX
- Identificativo umano
- Tipo
- Titolo
- Descrizione
- Stato
- Livello di riservatezza
- Autore
- Data creazione
- Data ultima modifica
- Coordinate
- Metadati
- Relazioni
- Allegati
