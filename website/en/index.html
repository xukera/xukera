The vision

Knowledge is not stored. Knowledge is connected.

Xukera is not a CMS, not a WordPress plugin, and not a vertical application. It is a universal knowledge framework designed to represent entities, connections, metadata and meaning.

Knowledge becomes powerful when relationships emerge.
Architecture

Small core. Clear modules. Infinite applications.

Core

Node, Relation, Graph and Query form the foundation of the knowledge model.

Storage

Save and load graphs through interchangeable storage adapters.

Serialization

Export knowledge into readable and portable formats such as JSON.

Validation

Check the integrity of graphs before storing or using them in applications.

Query Engine

Filter nodes, traverse relations and discover paths across the graph.

Applications

Build digital archives, research tools, knowledge bases and investigative systems.

Code

Readable by design.

xukera.php
use Xukera\Core\Graph;
use Xukera\Core\Node;
use Xukera\Core\Query;

$graph = new Graph();

$graph->addNode(new Node(
    'villa_widmann',
    'place',
    'Villa Widmann',
    ['city' => 'Mira']
));

$results = Query::create($graph)
    ->whereType('place')
    ->whereMetadata('city', 'Mira')
    ->get();
